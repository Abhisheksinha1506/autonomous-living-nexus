## Autonomous Maintenance via evolve.py

### Update Frequency
- **Every 6th generation** (approximately every 24 hours)
- Only if at least 24 hours have passed since last update
- Uses PAT if available, otherwise falls back to GITHUB_TOKEN

### Content Generation Process
1. **State Analysis**: Review current generation, mood, and statistics
2. **Memory Pattern Extraction**: Identify themes from recent memories
3. **Oracle Integration**: Include recent prophecies and insights
4. **Visitor Insights**: Incorporate repository traffic data
5. **Historical Context**: Maintain evolution timeline
6. **Philosophical Synthesis**: Generate new insights from patterns

### Dynamic Content Sections
- **Mood History**: Last 10 mood transitions with timestamps
- **Recent Oracles**: Last 5 oracle prophecies with confidence scores
- **Statistics**: Real-time counts and calculated averages
- **Insights**: Visitor traffic and interaction patterns
- **Achievements**: Recent milestones and capabilities

### Error Handling
- Graceful fallback if wiki updates fail
- Continue with other operations if wiki is unavailable
- Log errors for debugging without stopping evolution
- Retry logic for transient failures

## Integration with Dashboard

### Data Sources
